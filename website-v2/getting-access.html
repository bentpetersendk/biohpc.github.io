Onboarding

Getting Access

Access starts with a PI-owned Project Space. Researchers and collaborators request membership in that Project Space after the PI has registered.

Access workflow

1

PI registration

The PI submits registration details for eligibility verification.

2

BioHPC verifies eligibility

BioHPC reviews the request against the current service scope.

3

Project Space request

The PI requests the shared environment for a research project.

4

User requests

Researchers request access to the specific Project Space they need.

5

Approval and provisioning

The PI or authorized Technical Contact approves, then BioHPC provisions access.

Request forms

All onboarding requests use the existing Airtable forms.
