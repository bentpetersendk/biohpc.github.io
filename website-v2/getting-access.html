Getting Access

BioHPC access is organized through PI-owned Project Spaces. Principal Investigators create Project Spaces for their research activities, and researchers, students, and technical staff request membership in those Project Spaces.

Access workflow

1

PI registration

The Principal Investigator registers with BioHPC and provides information required for onboarding.

2

BioHPC onboarding review

BioHPC reviews the registration request and confirms that the requested use falls within the current service scope.

3

Project Space request

The PI requests one or more Project Spaces for specific research activities.

4

User requests

Researchers request membership in the specific Project Space they need.

5

PI / Technical Contact approval

The responsible PI or an authorized Technical Contact reviews and approves membership requests.

6

Access provisioned

BioHPC provisions project storage, access to shared compute resources, and login permissions for approved users.

Additional Information

Technical Contacts

Principal Investigators may designate one or more Technical Contacts, such as bioinformaticians, research software engineers, or laboratory staff.

Technical Contacts can assist with onboarding, membership approvals, and routine administration while the PI remains the responsible owner of the Project Space.

Computerome migrations

Research groups migrating from Computerome can receive assistance with Project Space setup, onboarding, user migration, and data transfer to BioHPC.

Planning a migration?

Contact BioHPC before submitting your request so we can help coordinate data transfer, user migration, and onboarding.

Questions before applying?

If you are unsure whether BioHPC is suitable for your project, have questions about onboarding, Project Spaces, shared compute access, or migration from Computerome, please contact the BioHPC team.
