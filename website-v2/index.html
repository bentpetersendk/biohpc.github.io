What BioHPC provides

BioHPC combines compute, storage, software environments, and support for research groups working with biological data.

High-performance computing

Shared compute resources for analyses that are too large for laptops or local workstations.

Shared project storage

Storage organized around research projects so teams can work from common data and results.

Slurm job scheduling

Queued batch jobs help researchers run repeatable workloads and make fair use of resources.

Bioinformatics environments

Software, modules, and workflow support for common life science computing needs.

Research computing support

Guidance for onboarding, job submission, data organization, and troubleshooting.

Collaborative infrastructure

Project Spaces bring users, storage, accounting, and approvals together under PI ownership.

Who can use BioHPC

BioHPC supports biological research teams with a clear PI-owned structure for students, researchers, technical contacts, and approved collaborators.

Principal Investigators

PIs own Project Spaces and remain responsible for membership, usage, and compliance.

Research groups

Groups can organize compute and data around shared scientific work.

Students and researchers

Postdocs, PhD students, and Master's students join approved Project Spaces.

Bioinformaticians

Technical users can support workflows and help teams use the infrastructure effectively.

Technical Contacts

Delegated contacts can help review requests and support day-to-day administration.

Approved collaborators

External collaborators may be added when the PI approves their participation.

How BioHPC works

Access follows the established Airtable-based onboarding workflow: PI registration, Project Space request, user request, approval, and provisioning.

1

PI registration

The PI registers so BioHPC can confirm eligibility and identify the responsible owner.

2

Project Space request

The PI requests one or more Project Spaces for specific research activities.

3

User request

Researchers request access to the relevant Project Space rather than joining BioHPC directly.

4

PI approval

The PI or an authorized Technical Contact reviews and approves membership.

5

Access provisioned

BioHPC provisions compute, storage, and login access for approved users.

Project Spaces are the organizing unit

Users join Project Spaces. Each Project Space groups the research environment, membership, accounting, and collaboration under PI responsibility.
