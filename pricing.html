Usage and accounting

Pricing and Resource Usage

BioHPC is a Department of Biology initiative that provides Department of Biology employees with access to shared compute and storage resources through approved Project Spaces.

Compute usage is charged according to the Rubus cost model, while storage follows the University of Copenhagen storage pricing model.

Compute usage is tracked through the Rubus accounting system, while storage allocation is organized through Project Spaces. BioHPC can provide usage summaries and reports to help approved Department of Biology Principal Investigators understand resource consumption and project costs.

Compute Pricing

Compute charges are based on the highest of CPU usage or memory allocation.

Current rates

  • 1 CPU hour = 0.06 DKK
  • 4 GB memory allocated for 1 hour = 0.06 DKK

The higher value determines the charge.

Important billing rule

Jobs are billed according to whichever resource contributes the highest cost:

  • CPU allocation
  • Memory allocation

This ensures that both CPU-intensive and memory-intensive workloads are accounted for fairly.

Example 1

2 CPUs
8 GB memory
10 hours

CPU cost:
2 x 10 x 0.06 DKK = 1.20 DKK

Memory cost:
(8 GB / 4) x 10 x 0.06 DKK = 1.20 DKK

Charge: 1.20 DKK

Example 2

8 CPUs
16 GB memory
24 hours

CPU cost:
8 x 24 x 0.06 DKK = 11.52 DKK

Memory cost:
(16 GB / 4) x 24 x 0.06 DKK = 5.76 DKK

Charge: 11.52 DKK

CPU dominates.

Example 3

4 CPUs
128 GB memory
12 hours

CPU cost:
4 x 12 x 0.06 DKK = 2.88 DKK

Memory cost:
(128 GB / 4) x 12 x 0.06 DKK = 23.04 DKK

Charge: 23.04 DKK

Memory dominates.

Typical genomics job

16 CPUs
64 GB memory
48 hours

CPU cost:
16 x 48 x 0.06 DKK = 46.08 DKK

Memory cost:
(64 GB / 4) x 48 x 0.06 DKK = 46.08 DKK

Charge: 46.08 DKK

CPU and memory contribute equally.

Storage Pricing

Storage pricing follows the University of Copenhagen storage model.

Current UCPH prices for 2026

  • 980 DKK per TB per year with backup and disaster recovery
  • 421 DKK per TB per year without backup and disaster recovery

Storage prices are maintained centrally by UCPH and may change annually.

View UCPH Storage Pricing

Storage costs are separate

Storage pricing follows the University of Copenhagen storage model and is independent of compute consumption.

Storage allocation is organized through approved Project Spaces so storage usage and associated costs can be understood at project level.

Storage costs may become relevant when budgeting large sequencing datasets, imaging projects, long-term archives, or Department of Biology employee research projects.

Understanding Resource Usage

Compute usage

Compute usage is tracked per user (KU-ID) through the Rubus accounting system.

BioHPC can provide usage reports to approved Department of Biology Principal Investigators showing compute consumption associated with their Project Spaces and approved Department employees.

Storage usage

Storage allocation is organized through Project Spaces.

This makes it possible to understand storage consumption and costs at the level of individual research projects.
